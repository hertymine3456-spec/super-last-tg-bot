# Telegram Suggestion Bot

Простой Telegram-бот для поддержки группы предложений и канала публикации.

## Что умеет бот

- Пересылает входящие личные сообщения в группу предложка.
- Позволяет публиковать сообщения из группы в канал через `/publish`.
- Ведёт лог публикаций, банов, разбанов и варнов.
- Поддерживает предупреждения (`warn`) с учётом количества.
- Блокирует спам и хранит состояние в JSON-файлах.
- Автообновляет `GROUP_ID` из `.env` и сохраняет новый ID при смене группы.

## Настройки в `.env`

- `BOT_TOKEN` — токен бота.
- `OWNER_ID` — ID владельца бота.
- `CHANNEL_ID` — ID или юзернейм канала.
- `GROUP_ID` — ID или юзернейм группы предложка.
- `LOGS_FILE` — имя файла логов (по умолчанию `logs.json`).
- `ALLOWED_FILE` — имя файла разрешённых пользователей (по умолчанию `allowed_users.json`).
- `AUTO_UPDATE_GROUP_ID` — включить автоматическое обновление `GROUP_ID` при смене группы (`1` или `0`).

## Установка

1. Установите Python 3.10+.
2. Создайте и активируйте виртуальное окружение.
3. Установите зависимости:

```powershell
python -m pip install -r requirements.txt
```

4. Создайте `.env` на основе `.env.example` и заполните данные.
5. Запустите бота:

```powershell
python bot.py
```

## Как настроить

1. Добавьте бота в группу предложка и в канал.
2. Сделайте бота админом в группе и канале.
3. Верните владельца/админов в `allowed_users.json` или используйте `/allow <user_id>`.
4. Если вы меняете группу, оставьте `AUTO_UPDATE_GROUP_ID=1` для автоматической подстройки.

## Как пользоваться

- Пользователь отправляет боту личное сообщение.
- Бот пересылает сообщение в группу предложка и сообщает: `сообщение было отправленно`.
- Админ отвечает на пересланное сообщение в группе командой `/publish`.
- Бот копирует сообщение в канал и сохраняет лог.

## Команды

- `/start` — приветствие и информация о боте.
- `/help` — список доступных команд.
- `/publish` — переслать сообщение из группы в канал.
- `/logs` — показать логи (только владелец и разрешённые пользователи).
- `/logs_week` — показать логи за последние 7 дней.
- `/logs_month` — показать логи за текущий месяц.
- `/warn` — выдать предупреждение пользователю.
- `/warns` — показать предупреждения пользователя.
- `/clearwarns` — очистить варны пользователя.
- `/ban` — забанить пользователя.
- `/unban` — разбанить пользователя.
- `/banned` — показать список забаненных.
- `/allow <user_id>` — разрешить доступ к логам.
- `/revoke <user_id>` — заблокировать доступ к логам.
- `/allowed` — показать список разрешённых пользователей.

## Файлы проекта

- `bot.py` — основной код бота.
- `config.py` — загрузка `.env` и конфигурация.
- `README.md` — инструкция.
- `requirements.txt` — зависимости.
- `warns.json` — хранение предупреждений.
- `banned_users.json` — хранение забаненных пользователей.
- `logs.json` — журнал событий.
- `allowed_users.json` — разрешённые пользователи.
- `reply_map.json` — карта пересланных сообщений.

## Советы

- Всегда держите `AUTO_UPDATE_GROUP_ID=1`, если меняете группу.
- Если варны не видны, проверьте доступы и статус пользователя в `allowed_users.json`.
- В личке бота используйте `/logs`, `/logs_week` и `/logs_month` для проверки.
